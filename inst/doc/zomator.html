Intro to the zomator package

A package that using API to access to Zomato.

Installation

You can install the stable version from CRAN, when available

install.packages("zomator")

Or the development version from Github

install.packages("devtools")
devtools::install_github("yitang310/zomator")

Then load the package into the R sesssion.

library("zomator")

Usage

Inside zomator, there are 12 functions:

get_categories, get_cities, get_collections, get_cuisines, get_establishments, get_geocode, get_location_details, get_locations, get_dailymenu, get_restaurant, get_reviews, get_search.

API requests to Zomato now require users to provide an API key, which can be obtained on Zomato.

You can pass it to the 'key' argument in zomator functions, but we highly recommend storing it as an environment variable called API_KEY in your file.

You can keep your .Renviron file in your global R home directory (R.home()), your user's home directory (Sys.getenv("HOME")), or your current working directory (getwd()). Remember that Renviron is loaded once when you start R, so if you add your API key to the file you will have to restart your R session. See https://csgillespie.github.io/efficientR/r-startup.html for more information on R's startup files.
